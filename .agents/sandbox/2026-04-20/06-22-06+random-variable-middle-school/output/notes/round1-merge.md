# Round 1 Merge: Random Variable Lesson

## Target

- Audience: middle-school students with zero probability background
- Language: English
- Format: short TeX/PDF handout
- Constraint: page 1 should be understandable with only middle-school math

## Strongest Agreements

1. Start from a familiar uncertain situation, not from notation.
2. Keep the first definition as:
   "A random variable is a rule that gives a number to each possible outcome of a random experiment."
3. Explicitly separate:
   - the random situation,
   - the outcome that happens,
   - the number assigned to that outcome.
4. Explain time order:
   - before the experiment, the value could be several numbers;
   - after the experiment, it becomes one definite number.
5. Delay heavy notation and avoid measure-theory language entirely.
6. Use at least one non-example.

## Best Candidate Examples

1. One die roll:
   - simplest start;
   - shows a case where the outcome is already a number.
2. One coin flip with coding:
   - Heads -> 1, Tails -> 0;
   - shows that the number can be assigned by choice.
3. Two coin flips with X = number of heads:
   - HH -> 2, HT -> 1, TH -> 1, TT -> 0;
   - best example for showing that different outcomes can give the same random-variable value.
4. Optional extension:
   - weather coded as numbers, if we want one example whose outcomes are words.

## Main Misconceptions To Address

1. A random variable is just "a random number."
2. The outcome itself is the random variable.
3. The rule creates the randomness.
4. One experiment has only one possible random variable.
5. Outputs do not need to be numbers.
6. One outcome can be assigned more than one value.

## Proposed Writing Direction

1. Real-life question
2. New explanation principles
3. Beginner questions
4. Intuition and first definition
5. Small examples in increasing depth
6. Very light notation
7. Non-examples and common mistakes
8. Tiny exercises
9. Final summary

## Likely First-Page Assets

- A one-line diagram:
  random experiment -> outcome -> rule -> number
- A tiny die table
- A two-coin table

## Open Decisions For Round 2

1. Is the die example too trivial to be first, or is it the right low-friction opening?
2. Should the weather example stay, or does it risk suggesting arbitrary coding without meaning?
3. Should page 1 include notation like X and X = 3, or delay that to page 2?
4. How much probability notation, if any, should appear in this note?
5. What theorem/fact is worth including for a middle-school audience without overloading the lesson?
