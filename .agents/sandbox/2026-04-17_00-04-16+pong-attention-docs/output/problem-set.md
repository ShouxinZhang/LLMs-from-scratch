# Low-Pong 核心问题集

只保留后续实现和验收真正需要回答的问题。

## Q1. 最小任务是什么

定义一个 `Low-Pong`：

- 单球、单挡板
- 离散时间
- 球每步向左移动 1 格
- 纵向速度为 `-1` 或 `+1`
- 不考虑顶底反弹
- 观测只有 `(x_t, y_t, p_t)`

## Q2. 为什么当前帧不够

要给出一个严格反例：

- 两段历史有相同当前帧
- 但最优动作不同

这一步是 attention 出场的前提。

## Q3. 最少需要几帧

在核心简化任务里，要证明：

- 一帧不够
- 两帧足够

并给出显式最优策略。

## Q4. 最优动作到底依赖什么

不要直接问“上还是下”，而是先问：

\[
y_{\text{hit}} = y_t + x_t v.
\]

也就是球最终会在什么位置撞到挡板。

动作只是：

\[
\operatorname{sgn}(y_{\text{hit}} - p_t).
\]

## Q5. attention 到底要检索什么

在这个最小任务里，relevant history 只有：

- 当前帧 `o_t`
- 上一帧 `o_{t-1}`

所以 attention 的核心职责不是“总结全历史”，而是“把这两个 token 抓出来”。

## Q6. 为什么必须是 causal attention

因为当前动作只能依赖过去历史。

如果训练时允许访问未来帧，就会发生信息泄漏，任务定义本身被破坏。

## Q7. 需要证明什么

最终只需要证明三条：

1. 不存在全局最优的单帧策略
2. 存在全局最优的两帧策略
3. 存在 causal multi-head attention 策略，可以精确实现该两帧最优策略
